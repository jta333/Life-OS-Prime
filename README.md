# LIFE OS PRIME

A luxury-class AI-powered personal performance dashboard, habit tracker,
routine optimizer, energy analyst, goal architect, and AI coach in one
Next.js app.

> Track daily routines · analyze habits · find leverage · build elite systems ·
> measure consistency · improve focus · reduce wasted time.

---

## Stack

- **Next.js 15** (App Router) + **TypeScript** + **React 19**
- **Tailwind v4** (CSS-first) + **shadcn-style** UI primitives
- **Supabase**, Postgres + Auth + RLS
- **Anthropic SDK**, Claude Haiku (chat) + Opus (deep weekly/monthly reviews) with
  prompt caching on the persona system prompt
- **Recharts** for charts · **date-fns** · **next-themes** · **sonner**

---

## Running locally

Requires Node.js LTS (v20+). Use **npm** (see `CLAUDE.md` for why npm over pnpm):

```bash
npm install
npm run dev
```

Open <http://localhost:3000>. The app runs out-of-the-box on **demo data** -
no env vars required.

### Going live with Supabase + Claude

1. Create a Supabase project.
2. Apply the migration in `supabase/migrations/0001_init.sql`
   (via Supabase Studio SQL editor or the CLI).
3. Copy `.env.example` → `.env.local` and fill in:

```bash
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=...
SUPABASE_SERVICE_ROLE_KEY=...
ANTHROPIC_API_KEY=sk-ant-...    # optional but recommended
```

4. Restart the dev server. Sign-up will now create a real user, onboarding
   persists, the AI coach streams real responses, and weekly/monthly reviews
   are generated by Claude.

If `ANTHROPIC_API_KEY` is missing the dashboard still works, scoring is
deterministic and the coach gracefully falls back to a static principle.

## Architecture notes

- **Single source of truth for the persona** lives at `src/lib/persona.ts` and
  is injected into every Claude call as a cached system block.
- **Deterministic scoring** (`src/lib/scoring.ts`) computes all 8 scores
  without an LLM, the dashboard renders instantly. AI is reserved for
  narrative insights.
- **Demo mode** (`src/lib/demo-data.ts` + `src/lib/data.ts`) auto-activates when
  Supabase isn't configured, so cloning the repo gives an immediately
  interactive preview.
- **RLS** is enforced via `user_id = auth.uid()` policies on every table.
- **Middleware** (`middleware.ts`) gates authed routes and redirects
  unauthenticated traffic to `/sign-in`. With no Supabase, middleware no-ops.
- **Prompt caching** is enabled via `cache_control: ephemeral` on the persona
  system block to keep per-message coach cost low.
